Arquitetura da aplicação

Como organizar os dois modos (comparação + calculadora individual)

A

Abas na mesma página

Um único index.html com três abas: Comparar | Calcular | Histórico. Tudo carrega junto, sem navegação.

Prós

  • Mais simples de implementar
  • Mantém estado entre abas
  • Ideal para mobile (PWA)

Contras

  • HTML inicial maior
B

Múltiplas páginas HTML

index.html (comparação) + calculadora.html + historico.html — páginas separadas com links entre si.

Prós

  • Separação clara
  • URLs distintas por seção

Contras

  • Perde estado ao navegar
  • Mais arquivos para manter
C

SPA com hash routing Recomendado

Página única com rotas via #/comparar, #/calcular/shopee, #/historico. Mantém estado, URLs compartilháveis e PWA perfeito.

Prós

  • URLs por seção (bookmarkável)
  • Estado global preservado
  • PWA-friendly
  • Mais escalável

Contras

  • Ligeiramente mais complexo
