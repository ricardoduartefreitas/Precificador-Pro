Design — Arquitetura e Estrutura

Organização modular dos arquivos da nova versão

Estrutura de arquivos
precificador-marketplace/
├── index.html ← entrada única da SPA
├── style.css ← tema dark + componentes
├── manifest.json ← PWA instalável
├── sw.js ← service worker (offline)
│
├── js/
│ ├── router.js ← hash routing (#/comparar, #/calcular/:p, #/historico)
│ ├── state.js ← estado global (inputs, resultados, modo PRO)
│ ├── engine.js ← lógica de cálculo (independente de plataforma)
│ ├── auth.js ← freemium: contador + validação de senha PRO
│ ├── history.js ← salvar/listar/exportar CSV
│ └── ui.js ← renderização das 3 views
│
└── platforms/
├── mercadolivre.js ← taxas ML por categoria/nível
├── shopee.js ← taxas Shopee CNPJ/CPF/faixas
├── amazon.js ← taxas Amazon por categoria + FBA/FBM
├── tiktok.js ← taxas TikTok Shop
└── shein.js ← taxas Shein Marketplace

Princípios da arquitetura

  • engine.js não conhece plataforma nenhuma — recebe taxas como parâmetro e devolve resultado
  • platforms/*.js são arquivos de dados puros — atualizar taxas = editar um número em um arquivo
  • state.js mantém os inputs preenchidos ao trocar de aba — sem redigitar
  • router.js gerencia hash URL — o vendedor pode favoritar #/calcular/shopee
